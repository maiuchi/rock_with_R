4 Data analysis in R

4.1 R Studio

What is R Studio?

  • An Integrated Development Environment (IDE) that is built for R
  • There are a couple options for IDEs, but I would recommend R Studio if you do not have your preferred IDE yet.

4.2 Set up an environment

  1. Install R and R studio (if you haven't downloaded them yet)
    • R Studio
    • R
      • If you want to update R, download a newer version of R and close R Studio.
      • The newer version should be installed by the time you reopen your R Studio.

  2. Create an R project file (optional, but recommended)
    • Name your file: myFirstR
      • On Mac, go to File -> New Project -> New Directory -> New Project -> Directory Name -> Create Project
    • Benefits of making an R project file
      • If you work as a data analyst, you often have a couple of projects you are working on at the same time. Of course, you can run each R scripts on R without making a project file, but I find it hard to track data frames you create
      • Project files will keep you organized
      • Once you close your R studio, all the temporaly data in the Global Environment that are not saved will be gone. It will be a good reminder for you to clean the code and make sure things work before closing it.

  3. Create an R script file
    • Name your file: r_workshop
      • Yes, you can type your code directly to Console and run. However, unless it's a quick check, I rarely type my code directly to Console.
      • For Mac users, "Command" and "Return" to run your code on a script
      • For Windows users, "Control" and "Enter" to run your code on a script

  4. Check and set up your work directory
    • Work directory: a place/ folder on your computer where your files for the project locate
#check your work directory
getwd()

#set your work directory 
setwd("/YOUR FILE PATH HERE/")

#Mac users (if you made a folder on your desktop)
setwd("/Users/your-user-name/Desktop/myFirstR")

#PC users (if you made a folder on your desktop)
setwd("C:\\Users\your-user-name\Desktop\myFirstR")

4.3 R packages

What are R packages?

  • "A package is essentially a library of prewritten code designed to accomplish some task or collection of tasks" (Lander, 2014)
  • This is what makes R great (I think...). You can, for instance, run a linear regression model using stats package, visualize data using ggplot2 package etc...
  • Packages I use often and some good ones I've heard:
    • ggplot2 - data visualization
    • tidyverse - data wrangling
    • dplyr - data wrangling
    • tidyr - data wrangling
    • stringr - dealing with characters
    • lubridate - dealing with dates
    • formattable - producing nice-looking tables
    • arsenal - stats calculation

How do you use packages in R?

  • First, you need to install packages if you've never installed them before
  • Load packages anytime you open a new project file or restart R Studio
  • I suggest you to load all the packages you need for your analysis at the beginning of your R script
  • Question about packages and/ or functions?
    • Type in ?name-of-package or ?name-of-function() in Console. R will give you a help page
#install packages 
install.packages("name-of-a-package")
  
#load packages
library(name-of-a-package)

#call help page on dplyr
?dplyr

4.4 Oooops, my code is not working!

Life as a data analyst? Google is your BFF, because we google all the time! Well... we don't remember all the codes/ commands, right? Don't be afraid to google! Don't feel bad that you don't know commands or you are getting stupid error messages. It happens all the time and is just normal!

  1. Google
    • You can copy + paste error messages from your R Console
    • Type brief description of your issue
      • (e.g.) merge columns in r
      • You will learn by practice how best to describe your issues to get what you are looking for
  2. Stack Overflow
    • My most used trouble shooting website
      • Not only for R, but many other languages
  3. Quick R
    • Good resource for beginners
